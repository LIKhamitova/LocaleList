# TASK_SPEC.md
## Цель и желаемый результат
Разработать веб-приложение LocaleList, которое отображает таблицу локалей мира.
На данный момент отображается 15 самых популярных локалей.  
В каждой строке: код локали (en-AU), язык + страна, валюта + TLD, флаг страны, таймзона + столица.  
Данные приходят с Node-сервера (статический JSON / in-memory), UI — чистый HTML/CSS/JS без фреймворков.
Дизайн простое - черное на белом
Для валют показывается код и символ.
## Текущее поведение и желаемое
Проекта нет, src/ пуст.  
Желаемое: открываешь index.html → видишь таблицу с 5 колонками и набором стран.
## Границы задачи (scope и non-goals)
**Входит:**
- сервер на Node (без npm-зависимостей), отдающий JSON по GET
- клиент: index.html + style.css + app.js
- флаги — inline SVG-строки в данных (без внешних файлов)
- одна страница, никакого роутинга
  **НЕ входит:**
- поиск, фильтры, сортировка
- пагинация
- БД / файлы с флагами на диске
- сборщики (webpack, vite), фреймворки (React/Vue), npm-зависимости
- тесты (кроме ручной проверки в браузере)
## Ограничения и границы изменений
- Node.js из системы, без `package.json` зависимостей — только `http`/`fs`/`path` (core modules)
- `src/` — корень клиента: `index.html`, `style.css`, `app.js`
- `src/server/` — серверный код: `server.js`, `locales.js` (данные)
- Данные живут в JS-объекте, читаемом и сервером, и клиентом через один JSON-файл
## Пакет доказательств
Ответ на каждый milestone (см. ниже) подтверждается скриншотом таблицы в браузере.
## Критерии приёмки
1. `node src/server/server.js` → сервер слушает порт, без ошибок
2. `curl http://localhost:XXXX/locales` → валидный JSON с массивом локалей
3. Открытие index.html (через сервер) → таблица с 5 колонками, заполненными данными
4. В таблице не меньше 10 строк
5. Все флаги отображаются (SVG inline)
6. Вёрстка на мобильном экране (±375px) не разваливается
## План проверки
1. Запустить сервер, проверить консоль
2. curl /locales — проверить структуру и количество записей
3. Открыть в Chrome, проверить таблицу визуально
4. Уменьшить ширину окна до 375px — проверить отзывчивость
## Готовность задачи
Задача считается готовой, когда все критерии приёмки выполнены, и пользователь подтвердил, что таблица выглядит приемлемо на его экране.
## Ожидаемый порядок работы
Milestone 1 → Milestone 2 → Milestone 3, последовательно.
## Ожидаемый артефакт
Работающее веб-приложение: файлы в `src/`, сервер запускается одной командой.
