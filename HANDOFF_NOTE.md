# Milestone 2 — Handoff Note

## Что сделано

Реализован пользовательский интерфейс для просмотра списка локалей, получаемых из API Milestone 1.

### Созданные файлы

| Файл | Назначение |
|---|---|
| `src/index.html` | HTML-каркас: таблица с 5 колонками, заголовок, индикатор загрузки и блок ошибки |
| `src/style.css` | Чёрно-белый дизайн, чередование строк, адаптивность под мобильные (375px) |
| `src/app.js` | Клиентский JS: fetch `/locales`, рендер таблицы, экранирование XSS, обработка ошибок |

### Изменённые файлы

| Файл | Что изменено |
|---|---|
| `src/server/server.js` | Добавлена раздача статики (`/` → `index.html`, `/style.css`, `/app.js`). Валидация расширений и защита от path traversal. Контракт `/locales` не изменён. |

## Архитектура

- Сервер раздаёт статику из `src/` (на уровень выше себя)
- Клиентский JS делает `fetch('/locales')` на тот же origin — никаких CORS-проблем
- SVG-флаги вставляются через `innerHTML` (данные приходят уже как строка `<svg>...</svg>`)
- Все текстовые поля экранируются через `esc()` перед вставкой

## Как запустить

```bash
node src/server/server.js
# → http://localhost:3000
```

## Проверка

```bash
# 1) Тесты M1 не сломаны
node test/test-locales.js

# 2) Статика отдаётся
curl http://localhost:3000/           # → HTML
curl http://localhost:3000/style.css  # → CSS
curl http://localhost:3000/app.js     # → JS

# 3) API не изменился
curl http://localhost:3000/locales    # → JSON, 15 записей

# 4) 404 на неизвестные пути
curl http://localhost:3000/whatever   # → {"error":"Not found"}
```

## Scope notes

- Контракт `/locales` не изменялся — обратная совместимость с M1 сохранена
- Нет npm-зависимостей (только `http`, `fs`, `path`)
- Нет фреймворков (чистый HTML/CSS/JS)
- Нет внешних файлов флагов (SVG inline из данных)
- Нет поиска/фильтров/пагинации — выходят за scope M2
